# Render Spec

The final delivery pipeline owns locked assets, final readable text, and QA. Use `scripts/run_cover_pipeline.py` for delivery. Use `render/render_cover_pillow.py` only through the pipeline. Use `render/render_cover.ts` only for optional preview.

## Inputs

- `sidecar.json`: variable cover content.
- `layout_contract.json`: locked platform geometry and typography.
- `template_registry.json`: color tokens only, not geometry.
- `background.png`: image2-generated textless background. Required for final delivery. No placeholder or MVP background is supported.

## Output Location

- Default output root is `./output/social-media-cover/`, as defined in `config/global_defaults.json`.
- Each run should write into `./output/social-media-cover/<run_id>/`.
- Keep `background.png`, `{platform}_final_pillow.png`, sidecar JSON, and trace JSON in the same run folder.
- Avoid sibling folders named `social-media-cover_<topic>` directly under `output/`.
- `scripts/run_cover_pipeline.py` creates `run_manifest.lock.json` before rendering. The renderer and QA must read this manifest; do not let later steps reselect template, palette, fonts, or output path.

## Rendering

- Final path: use `scripts/run_cover_pipeline.py`, which calls `render/render_cover_pillow.py` with a required manifest and then calls blocking QA.
- Preview path: use HTML/CSS absolute positioning and Playwright screenshot only after a real sidecar and real background exist. It is not a fallback output and cannot be delivered as final.
- If `{platform}_final_pillow.png` exists, it is the authoritative final image for delivery and preview. Do not present an older `{platform}_final.png` or inline preview as the final cover.
- Load the image2 background as an image layer for final delivery. Do not generate or use any template background or placeholder scene when a real background is missing.
- Render title color from the manifest/global defaults only. Default locked colors are title `#FFD61E` and subtitle `#FFFFFF`; do not infer text colors from platform, template mood, or background.
- Load the explicit configured font path. If the locked font is missing, fail instead of falling back silently.
- Render title and subtitle as one locked visual group. In Douyin mode the group is vertically centered by equal visual margins inside the title-safe zone defined in `layout_contract.json`.
- In Douyin mode, use the no-stroke/no-shadow values in `layout_contract.json`; readability must come from the image2-generated title buffer. Do not add a dark title package, hard frame, CSS text shadow, synthetic offset redraw, or fake bold duplicate pass unless `layout_contract.title_package.backing` explicitly enables a diagnostic backing.
- In WeChat mode, render title and subtitle as a left-aligned group vertically centered on the 21:9 canvas. Use the WeChat font sizes and stroke values in `layout_contract.json`, not the Douyin title scale; default WeChat stroke/shadow should be light or disabled because background readability is planned upstream.
- The image2 background should keep the title-safe zone dark and low-detail. It must not place yellow glow washes, large UI panels, glass frames, title boxes, or hard-edged containers behind the future text.
- For WeChat, the left title side may contain subtle article-derived background elements generated by image2, but they must not form a visible rectangle or visual frame around the title.
- Do not add an extra hard dark compositor backing by default, because it can muddy the image2-designed title area. If cleanup is needed, use only a soft theme-matched glass/light zone with weak edges.
- Apply fixed `font-size`, fixed `font-weight`, fixed `line-height`, and `white-space: nowrap`. Use stroke/shadow only as defined in `layout_contract.json`.
- Screenshot the fixed viewport with Playwright.
- Do not use decorative lines or accents that can touch the first title character.
- Avoid solid, dead-feeling borders around the title area. Frames should be weakened and subordinate to the final text hierarchy.
- If Playwright/CSS text appears fuzzy or stroke-eaten at 100% inspection, do not accept it as final. Use Playwright only as a layout preview and switch final text compositing to SVG/resvg or Canvas/Pillow using the same `layout_contract.json` coordinates.

## Blocking QA

Run `scripts/run_cover_pipeline.py`; it runs `scripts/qa_cover.py` after rendering. The QA report and pipeline report must pass before delivery. QA checks final size, output root, template drift, palette drift, font drift, title overflow, title-safe-zone brightness, yellow saturation, edge density, large-frame likelihood, and semantic object lock.

## Extensibility

Future renderers may use SVG + resvg or Canvas/Pillow, but they must read the same `layout_contract.json` and pass the same QA checks. The final production renderer should prioritize crisp Chinese text over CSS convenience.
